Modern Web Architecture

So before we look the modern web Architecture. I'd like go over the where we came from, by looking at the evolution of Web Development.

So when I started web developing application development, we called Web Masters back in those days. So there was Static HTML and if you wanted to something dynamic like display web visit counter (remember those). Back when I was at Parktown Boys's I had created a website for the Geography department and I wanted to display a web visit counter, and that's when I discovered that the HTML part of the website just gives you a static stuff for dynamic stuff there was something I had to do on the server. When I investigated I advised to use some sort of CGI scripting. So after digging around the interwebs I found some script uploaded it and we had a web visit counter. So the evolution has been:

Web Frameworks

Convention Over Configuration

		+-----------------------------------+
		|         CLIENT - WEB BROWSER      |
		+---------------^-------------------+
				| [HTML,FORM-POSTS,AJAX]
		+---------------v-------------------+
		| Ruby on Rails (MVC), ASP.NET MVC  | [SERVER]
		+-----------------------------------+

DHH (David Heinemeier Hansson) changed the game created Ruby on Rails which based on MVC pattern, it was a very popular framework. Others followed like ASP.NET MVC, Codeigniter PHP MVC

Convention over configuration is a software design paradigm which seeks to decrease the number of decisions that developers need to make, gaining simplicity, and not necessarily losing flexibility. Before web frameworks It was chaos, people were using ASP.NET & PHP which ever way they wanted. So with this we able to create Web Application which started rivaling Desktop applications. Then mobile devices started become popular new problems arose, smaller screens, slower network speeds because 2g and 3g. Granted you could create responsive web applications we still had another needs. example wanting to have a native app for the mobile device but consuming the same data.

Now we have these JavaScript Front-End Frameworks (Angularjs, Emberjs) which are inspired by this mobile evolution. So whether your on Web or Mobile you want consume the same RESTful API, so more computation is being handled on the front-end and the communication back and forth is JSON.
